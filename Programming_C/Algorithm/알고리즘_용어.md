
1. 알고리즘 효율성 판단//컴퓨터 하드웨어, OS환경 배제하고 평가.
    1)시간 복잡도(T(N)): 알고리즘이 결과를 출력하는 연산횟수 측정.
    2)공간 복잡도(S(N)): 알고리즘이 사용하는 메모리의 양을 측정.// 입력 데이터에 따라 사용되는 메모리양. 입력 데이터가 아님.

2. 점근적 표기법(알고리즘 효율성을 분석.)// 중요하지 않은 상숫값OR낮은 차수의 항은 제외하고 가장 큰 영향을 미치는 최고차항만을 남겨 단순화.(연산횟수의 추이만 보는것이므로 그렇게 목메서 계산할 필요가 없으므르...):입력되는 데이터의 양이 늘어날때 걸리는 처리 시간.
    1)빅오 표기법: 알고리즘의 최악의 경우일때의 성능을 분석. **상한(upper bound)**제공.//가장 많이 사용되는 분석법
    2)빅오메가 표기법: 알고리즘의 최상의 경우일때의 성능을 분석. 하한(lower bound)을 제공.
    3)빅세타 표기법: 알고리즘 시간 복잡도에 대한 상한과 하한이 일치할때 사용.//까다로움

3. 빅오 표기법
	 

	1)  	
    
        표기법	  	 명칭 		 데이터(N)늘어날떄 걸리는 시간
	     O(1)  	 	상수시간   데이터가 1개든 1억 개든 연산횟수(시간 복잡도) 일정.
	     O(longN) 	로그 시간  데이터가 늘어나도 연산횟수 아주 조금 늘어남.
	     O(N) 	 	선형 시간  데이터에 개수만큼 시간 복잡도 증가.
	     O(N^2)    	2차 시간   데이터가 늘어나면  폭발적으로 늘어남.//O(NM) 같은 경우도 있음.
