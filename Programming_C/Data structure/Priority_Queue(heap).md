    0. **완전 이진트리** 기반.
        키값(key value): 환자의 위급함 정도(우선순위)

    1.조건:
     데이터 push/pop 해도 완전 이진트리 특성이 유지돼야함.

    2. 종류:
     2-1 MAX Heap: 부모노드의 키값>= 자식노드들 키값(우선순위가 높은 높은 큰값이 루트에 위치)// 중복 허용
        //MAX Heap을 차례로 pop하면 내림차순 형태.
     2-2 Min Heap: 부모노드의 키값<= 자식노드들 키값
     (우선순위가 높은 작은값이 루트에 위치)//중복 허용
     // Min heap을 차례롤 pop하면 오름차순 형태.

    3. 그래서 도대체 Priority Queue는 어따씀?:
    e.g.,)
     데이터를 내림차순으로 pop하고 싶을때 
     기초적인 array의 bubble sort로는 O(N^2) 연산 필요($\because N개 데이터일때 for문^2. )
     priority Queue는 O(N*logN)연산만 필요. 즉, 압도적인 효율 자랑하므로 Priority Queue를 쓰는것.
